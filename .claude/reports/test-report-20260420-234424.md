## 不合格・既知問題

### test_runner.py::test_スモークテスト（既存 slow テスト）

**テスト名:** `test_スモークテスト` (`tests/test_runner.py`)
**分類:** 既知問題（T4 以前から存在、T8 以降も未修正のまま持ち越し）
**期待値:** `sys.executable -p "import sys; sys.exit(0)"` で returncode == 0
**実際の動作:** Python インタープリタは `-p` オプションを認識しないため returncode == 2（`Unknown option: -p` エラー）
**原因:** `claude` CLI の呼び出し規約（`claude -p <prompt>`）を `sys.executable`（Python）で代替しようとしているが、Python に `-p` オプションは存在しない
**影響範囲:** slow テストのみ。fast path（82 件）への影響なし
**優先度:** 低（fast path が全件合格のため DoD に影響しない）
**推奨修正方針:** T8 の `TestSlowSmokeRealProcess` と同様に、プラットフォーム別ラッパースクリプト（Windows: `.bat`、Unix: `.sh`）を `tmp_path` に生成して代替する

## クロスプラットフォーム留意点

### pathlib.Path 統一

`grep -R "os.path" src/` の結果: **該当なし**。
`src/clade_parallel/` 全モジュールで `pathlib.Path` のみ使用されており、OS 依存のパス文字列操作は排除されている。

### Windows での subprocess.run タイムアウト挙動

現在の実装（`runner.py`）は `subprocess.TimeoutExpired` を捕捉して `timed_out=True` として扱う。
Windows では Unix のような `SIGKILL` が存在せず、タイムアウト後の子プロセス残存がリソース問題を引き起こす可能性がある。
plan-report の T5 注記では「`kill()` 相当の後始末」の実装が指示されていたが、テスト観点では `TimeoutExpired` ハンドリングが正しく機能することのみを確認済み。
実際の子プロセス kill 処理の完全性は **code-reviewer が T10 で精査すべき項目** として申し送る。

### CRLF/LF 警告

Windows 環境での git 操作では CRLF/LF の自動変換に関する警告が発生することがあるが、
`pyproject.toml` の設定（`text=True` での subprocess 出力取得）および実装ロジック上、改行コードに起因するテスト失敗は発生していない。
テスト実行時にも CRLF 関連のエラーは観測されなかった。

### Windows 実機での全テスト合格

本テスト実行環境: `platform win32, Python 3.10.6`
fast path 82 件が Windows 実機で全件合格。

## T10（code-reviewer）への申し送り事項

### 重点確認項目

1. **タイムアウト後の子プロセス残存（Windows）**
   `runner.py` で `TimeoutExpired` を捕捉後の子プロセス kill 処理について、Windows 固有の挙動（`SIGKILL` 非存在）を考慮した後始末が実装されているか確認。
   テストでは `timed_out=True` の設定のみ検証しており、実際の子プロセス終了保証までは確認していない。

2. **カバレッジ未充足行の実装品質**
   - `runner.py:L109-111` — タイムアウト後の kill/後始末コードパス
   - `runner.py:L239` — 未使用または到達困難なエラーハンドリングパス
   - `manifest.py:L132, L207-208, L211` — エッジケースのエラーパス
   - `cli.py:L155` — 到達困難な分岐（`SystemExit` 再送出パス等）
   これらの行が意図的に未到達なのか、テスト設計の漏れなのかを確認。

3. **疎結合の最終確認**
   `grep -R "import clade\|from clade" src/ tests/` はテスト済みだが、`pyproject.toml` のランタイム依存が `PyYAML` のみであることもソース側から確認を推奨。

4. **`_execute` 内部関数のスレッドセーフ性**
   `runner.py` の `_execute` 関数が `ThreadPoolExecutor` から複数スレッドで同時呼び出される際、引数として渡される `Task`（`frozen=True`）と `claude_exe`（文字列）が不変であることの確認。
   共有可変状態がスレッドをまたいで存在しないことを静的に確認されたい。

5. **slow テストの修正**
   `tests/test_runner.py::test_スモークテスト` は T4 以来の既知問題。T8 の `TestSlowSmokeRealProcess` と同様の方針（プラットフォーム別ラッパースクリプト使用）への修正を推奨。
