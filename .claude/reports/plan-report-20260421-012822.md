## developer への注意事項（T22 実行時）

### T22 E2E 手動確認の実施手順（推奨）

1. **`examples/manifest.md` の用意**（不在の場合のみ作成）
   - `clade_plan_version: "0.1"` を先頭に記述
   - `default.cwd: .` を設定
   - `tasks:` に 2 件以上（例: `code-reviewer` と `security-reviewer` を異なる `agent` として記述、それぞれ read-only 前提）
   - `read_only: true` を各タスクに明示（`false` は ManifestError となる）
   - 各タスクの `timeout_sec` は 60〜300 程度（短めを推奨）

2. **実行コマンド**
   ```bash
   clade-parallel run examples/manifest.md
   ```
   （`pip install -e .` でローカルインストール済みであることが前提）

3. **目視確認項目（ユーザー立会い）**
   - (a) `.claude/reports/code-review-report-*.md` と `.claude/reports/security-review-report-*.md` の **両方**が新規生成されていること（既存レポートとの区別はタイムスタンプで確認）
   - (b) CLI の終了コードが **0**（`echo $?` または `echo %ERRORLEVEL%`）
   - (c) 両レポートファイルのタイムスタンプがほぼ同時刻であり、直列実行より明らかに短時間で完了していること（並列実行されている証拠）
   - (d) CLI サマリ行がタスクごとに 1 行ずつ計 2 行出力され、それぞれに `ok` / `fail` / `timeout` のいずれかと `duration` が含まれていること

4. **結果報告**
   - 全 4 項目が確認できた場合: ユーザーに報告し `/end-session` を実行
   - 問題が発生した場合: developer が原因切り分け（manifest.md 不備 / 環境問題 / バグ）を行い、新規バグであれば **v0.2 の最優先修正**としてロードマップ追加

### 留意事項
- `claude` コマンドが PATH に通っていることが前提。未インストールであれば `--claude-exe` で明示パス指定
- `read_only: true` は **診断系エージェント（code-reviewer / security-reviewer / interviewer 等）に限定**して指定する。実装系エージェント（developer / architect など）は T22 の検証対象外
- 万が一子プロセスが残存した場合は、タスクマネージャ / `ps`・`pgrep` 等で確認し、v0.2 の「`except Exception:` 後始末」の必要性を実証する参考情報として記録
