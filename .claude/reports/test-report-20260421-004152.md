## 不合格・要修正（Red フェーズのため意図した失敗）

### F1: test_タイムアウト時に子プロセスがkillされる
**期待値:** `subprocess.Popen.kill()` が最低 1 回呼ばれ、`timed_out=True` の TaskResult が返る
**実際の動作:** `subprocess.Popen` が一切呼ばれない（`kill_call_count=0`）。現行実装は `subprocess.run` を使用しているため Popen インスタンスが生成されない
**再現手順:** `python -m pytest tests/test_runner.py::test_タイムアウト時に子プロセスがkillされる -v`
**優先度:** 高（T14 で F1 実装により Green 化が必要）

### F2: test_複数タスクがRunnerErrorを起こしても最初の1件のみ送出される
**期待値:** 2 件の `RunnerError` が raise されたとき、最初の RunnerError（"runner error #1"）が伝播される
**実際の動作:** 現行実装の `runner.py:L232-234` の `else: raise exc` により 2 件目の RunnerError（"runner error #2 for task 'task-b'"）が再送出される。最初のエラーは `runner_error` 変数に保持されるが実際には捨てられる
**再現手順:** `python -m pytest "tests/test_runner.py::test_複数タスクがRunnerErrorを起こしても最初の1件のみ送出される" -v`
**優先度:** 高（T15 で F2 実装により Green 化が必要）

### F3: test_危険な環境変数キーはManifestErrorを送出する（6 ケース）
**期待値:** LD_PRELOAD / LD_LIBRARY_PATH / LD_AUDIT / DYLD_INSERT_LIBRARIES / DYLD_LIBRARY_PATH / PYTHONPATH のいずれかを `task.env` に含むと `ManifestError` が送出される
**実際の動作:** 現行実装（`manifest.py:_parse_task`）は env キーを検証しないため `ManifestError` が送出されない（`DID NOT RAISE`）
**再現手順:** `python -m pytest tests/test_manifest.py -k "dangerous" -v` または `python -m pytest tests/test_manifest.py::test_危険な環境変数キーはManifestErrorを送出する -v`
**優先度:** 高（T16 で F3 実装により Green 化が必要）

## 追加したテストの詳細

### F1 向け新規テスト

`tests/test_runner.py` に以下を追加：
- `FakePopen` クラス（`communicate` が最初の呼び出しで `TimeoutExpired` を投げ、`kill()` のコールカウントを記録）
- `_TrackingFakePopen` クラス（モジュールレベルのリストにインスタンスを登録）
- `test_タイムアウト時に子プロセスがkillされる`: `monkeypatch.setattr("clade_parallel.runner.subprocess.Popen", _TrackingFakePopen)` で差し替え、`kill_call_count >= 1` を assert

### F2 向け新規テスト

`tests/test_runner.py` に以下を追加：
- `test_複数タスクがRunnerErrorを起こしても最初の1件のみ送出される`: `monkeypatch.setattr(runner_module, "_execute_task", fake_execute_task)` で `_execute_task` を直接置き換え、2 件の異なるメッセージの `RunnerError` を投げる。伝播されたエラーのメッセージが "runner error #1" であることを assert

### F3 向け新規テスト

`tests/test_manifest.py` に以下を追加：
- `test_危険な環境変数キーはManifestErrorを送出する`: `@pytest.mark.parametrize` で 6 ブロックキーを網羅（LD_PRELOAD / LD_LIBRARY_PATH / LD_AUDIT / DYLD_INSERT_LIBRARIES / DYLD_LIBRARY_PATH / PYTHONPATH）
- `test_通常の環境変数キーは許可される`: `FOO=bar` / `MY_VAR=hello world` で `ManifestError` が出ないことを確認（逆パターン）
