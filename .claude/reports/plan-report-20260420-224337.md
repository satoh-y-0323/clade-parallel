## developer への注意事項

**疎結合の厳守**
- `src/clade_parallel/` 配下のいかなるファイルでも `import clade` / `from clade import ...` / `import clade_*`（自パッケージ `clade_parallel` は当然可）を書かない
- 契約はマニフェスト（YAML フロントマター付き Markdown）とサブプロセス起動のみ

**クロスプラットフォーム**
- パス操作は `pathlib.Path` に統一。`os.path.join` / 文字列スライシング / `"/".join()` は原則禁止
- `subprocess.run` に渡す `cwd` は `Path` を渡して Python 側で吸収（Windows の区切り文字問題を avoid）
- 改行は LF（`newline="\n"`）でファイルを扱い、`text=True` で文字列として受け取る
- `os.environ` をコピーしてから `task.env` をマージ（`PATH` を消さない）

**コーディング規約**
- Black / Ruff 行長 88、ダブルクォート
- 全 public API に型注釈（`X | None` 構文）と Google Style docstring
- 例外は `ManifestError` / `RunnerError` を使用し、素の `Exception` を raise しない
- `frozen=True` の dataclass は `__post_init__` で正規化（`cwd` の `Path.resolve()` など）

**タイムアウト / race condition**
- `subprocess.run` の `timeout` を必ず指定し、`TimeoutExpired` を拾って子プロセスの後始末（`kill()` 相当）を明示的に行う
- `ThreadPoolExecutor` は必ず `with` 文で使用し、例外パスでも join されるようにする
- スレッド内で共有する可変状態を持たない（結果集約は `future.result()` 経由のみ）
- `run_manifest` の戻り値型 `TaskResult` / `RunResult` は `frozen=True` なので外部から書き換え不能

**approvals.jsonl から読み取った傾向**
- 上流レポート（requirements / architecture）はコメントなしで即承認されている → 設計方針は信任されている前提で忠実に実装する
- 否認履歴なし → 減点要素が無い実装を最初から目指し、承認コスト（再実装サイクル）を抑える

## tester への注意事項

- `subprocess.run` モックはプロジェクト共通フィクスチャ `fake_claude_runner(monkeypatch, outcomes)` として `tests/conftest.py` に配置し、T4 以降で全テストから再利用する
- **必ずカバーするシナリオ**:
  - 並列性（両タスクが別スレッドから呼ばれている / 総時間が直列合計より短い）
  - タイムアウト（`TimeoutExpired` 発火時の `timed_out=True`・他タスク継続）
  - 片方失敗（exit 1 / 例外 / タイムアウト）＋他方成功
  - マニフェスト異常系（未サポート version / 型エラー / `read_only: false` 混在 / 未知キー許容 / 空タスクリスト）
- 境界値は `@pytest.mark.parametrize` で網羅
- 実バイナリ代替スモークは `@pytest.mark.slow` で 1 本だけ保持し、CI の高速パスからは除外できるようにする

## reviewer への注意事項

**code-reviewer**
- 疎結合チェック（Clade import ゼロ）を静的に検証（`grep -R "import clade\|from clade" src/ tests/`）
- 依存追加の妥当性（`pyproject.toml` のランタイム依存が `PyYAML` 以外に増えていないか、dev 依存も妥当か）
- race condition の余地（共有可変状態・`ThreadPoolExecutor` の使い方・例外時のリソースリーク・`TimeoutExpired` 後の子プロセス残存）

**security-reviewer**
- `subprocess` を `shell=False`・リスト形式で呼んでいるか（コマンドインジェクション耐性）
- YAML は `safe_load` のみ使用しているか（任意オブジェクト構築禁止）
- `env` マージ時の機密情報の流出可能性と `PATH` 保持
- タイムアウト後の子プロセス残存による DoS リスク
- `--claude-exe` 任意指定による任意実行ファイル起動のリスク表記
