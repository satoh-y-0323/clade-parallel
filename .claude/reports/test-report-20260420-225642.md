## developerへの依頼事項（T3）
以下の公開 API を `src/clade_parallel/manifest.py` に実装すること:

- `ManifestError` — プロジェクト基底例外から派生した例外クラス
- `SUPPORTED_PLAN_VERSIONS = frozenset({"0.1"})` — サポートバージョン定数
- `Task` — `frozen=True` の dataclass（フィールド: `id`, `agent`, `read_only: bool`, `prompt: str`, `timeout_sec: int = 900`, `cwd: Path`, `env: dict[str, str]`）
- `Manifest` — dataclass（フィールド: `path: Path`, `clade_plan_version: str`, `name: str`, `tasks: tuple[Task, ...]`）
- `load_manifest(path: str | Path) -> Manifest` — YAML フロントマター付き Markdown を読み込み Manifest を返す関数

実装上の留意点:
- PyYAML の `safe_load` を使用すること（`full_load` / `load` は禁止）
- `read_only: false` のタスクは `ManifestError` を送出すること
- `cwd` の既定値はマニフェストファイルの親ディレクトリ (`Path.resolve()`)
- `timeout_sec` の既定値は 900 秒
- `env` の既定値は空 dict
- `prompt` の既定値は `"/agent-<agent>"` の形式
- 未知キーは無視すること
- フロントマター開始 `---` がない場合 / 閉じ `---` がない場合は `ManifestError`
- 空タスクリスト (`tasks: []`) は `ManifestError`
- `version_value` が文字列以外（int/float/None）の場合も `ManifestError`
