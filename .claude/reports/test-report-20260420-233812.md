## スモークテストの実装方針（シナリオ 5 補足）

plan-report の注記通り、`sys.executable -p <prompt>` 形式では Python が `-p` を認識しないため exit 2 となる。
対処として以下の方針を採用した:

- **Windows**: `fake_claude.bat` を `tmp_path` に生成。内容は `@echo off\r\nexit 0`。
- **Unix**: `fake_claude.sh` を `tmp_path` に生成。内容は `#!/bin/sh\nexit 0`、chmod 755。

いずれもランナーから `[exe, "-p", "<prompt>"]` の形で呼び出されるが、引数を無視して exit 0 を返す。
これにより実プロセス生成の E2E パスを検証できる。
