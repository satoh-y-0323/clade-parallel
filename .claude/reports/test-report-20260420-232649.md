## テスト設計のポイント

### モック戦略
- `monkeypatch.setattr(cli, "load_manifest", ...)` で `cli` モジュール内の参照名を差し替え
- `monkeypatch.setattr(cli, "run_manifest", ...)` で同様に差し替え
- `capsys` で stdout/stderr を捕捉
- `RunResult` / `TaskResult` はダミーインスタンスを組み立てて返す
- `ManifestError` / `RunnerError` は実例外を使用して raise

### テスト構成
- `TestMainExitCodes`: exit code 0/1/2/3 の検証（4件）
- `TestVersionFlag`: `--version` フラグ検証（1件）
- `TestHelpFlag`: `--help` フラグ検証（1件）
- `TestNoArgumentsError`: 引数なしエラー検証（1件）
- `TestQuietFlag`: `--quiet` フラグ検証（2件）
- `TestMaxWorkersPassthrough`: `--max-workers` 引数の透過検証（1件）
- `TestClaudeExePassthrough`: `--claude-exe` 引数の透過検証（1件）
- `TestSummaryOutput`: サマリー行フォーマット検証（4件、うち3件は `@pytest.mark.parametrize` で集約）

### 境界ケース
- `--version` / `--help` は `SystemExit` を発生させるため `pytest.raises(SystemExit)` で捕捉
- 引数なし呼び出しは `SystemExit` を発生させるため同様に `pytest.raises(SystemExit)` を使用し、`code != 0` のみを assert（argparse の exit 2 に拘束されない柔軟なアサーション）
