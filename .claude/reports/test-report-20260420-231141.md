## developerへの依頼事項（T5）

以下の公開 API を `src/clade_parallel/runner.py` に実装してください:

- `RunnerError` — 例外クラス（`FileNotFoundError` 時に raise）
- `TaskResult` — `frozen=True` dataclass
  - フィールド: `task_id: str` / `agent: str` / `returncode: int | None` / `stdout: str` / `stderr: str` / `timed_out: bool` / `duration_sec: float`
  - プロパティ: `ok: bool`（`returncode == 0 and not timed_out`）
- `RunResult` — `frozen=True` dataclass
  - フィールド: `results: tuple[TaskResult, ...]`
  - プロパティ: `overall_ok: bool`（全 `TaskResult.ok is True`）
- `run_manifest(manifest: Manifest | Path | str, *, max_workers: int | None = None, claude_executable: str = "claude") -> RunResult`
  - `Manifest` / `Path` / `str` の三態を受け付ける
  - `subprocess.run(cmd=[claude_exe, "-p", task.prompt], cwd=task.cwd, env={**os.environ, **task.env}, capture_output=True, text=True, timeout=task.timeout_sec, check=False)` を使用
  - `TimeoutExpired` → `timed_out=True` / `returncode=None`
  - 任意例外 → `stderr` にトレースバックを保存して他タスクを継続
  - `FileNotFoundError` → `RunnerError` として早期失敗
  - `ThreadPoolExecutor` を `with` 文で使用（`max_workers = max_workers or max(1, len(manifest.tasks))`）
  - **Clade モジュールを絶対に import しない**

### テスト実装の注意点（T5 developer 向け）

- 並列性検証テスト（`test_並列実行により総時間が直列合計より短くなる`）: 2 タスク各 0.2s sleep で総時間 < 0.35s を期待。CI の揺らぎを考慮してマージン 0.05s を確保済み
- スレッド検証テスト（`test_並列実行時に異なるスレッドから呼ばれる`）: 2 つの異なるスレッド ID が recorder に記録されることを検証
- `test_subprocess_runに正しい引数が渡される`: `call_args[0][0]` がコマンドリスト、`call_kwargs` に `cwd` / `env` が含まれることを前提
