# /prune-rules スキル

cluster-promote で昇格したルールを整理する。
孤立クラスタの削除と、重複・類似ルールの統合をユーザーと対話しながら実行する。

## 実行手順

### ステップ1: インベントリ収集

1. `Glob` で `.claude/instincts/clusters/*.json` を全件取得する
2. 各 JSON を Read し、`type === "rule"` のものを抽出する
3. 各ルールについて、`.claude/rules/{name}.md` の存在を Glob で確認する
4. 以下に分類する:
   - **正常**: clusters JSON とルールファイルの両方が存在する
   - **孤立クラスタ**: clusters JSON はあるがルールファイルが存在しない

`type === "rule"` のクラスタが0件の場合は「昇格済みルールが見つかりませんでした」と表示して終了する。

### ステップ2: 孤立クラスタをユーザーに確認する

孤立クラスタが存在する場合、以下の形式で提示する:

```
## 孤立クラスタ（ルールファイルが存在しません）

以下の clusters JSON に対応するルールファイルが見つかりませんでした:
- {name}: {summary}（昇格日: {promotedAt}）
...

clusters JSON を削除しますか？（残すと次回も同じ確認が発生します）
  [yes] 削除する
  [no]  スキップする
```

yes の場合: 対象の clusters JSON ファイルを Bash で削除する。

### ステップ3: 正常なルールファイルを読み込む

正常ルール（ルールファイルが存在するもの）を全件 Read する。

正常ルールが0件の場合は終了する。

### ステップ4: AI 分析

読み込んだルールを全件分析し、以下の観点で評価する:

- **統合候補**: 同じまたは近い内容をカバーしているルールのペア・グループ
- **問題なし**: 独立していて有効

### ステップ5: 分析結果をユーザーに提示する

以下の形式で提示する:

```
## /prune-rules 分析結果

対象ルール: N件

### 統合候補
[A] {rule-a} + {rule-b}
    理由: {重複・類似の根拠}
    提案: {rule-a} に統合し、{rule-b} を削除する

### 問題なし（変更不要）
- {name}: {summary}
...

対応する番号を入力してください（例: A）/ all / none
```

統合候補がない場合は「重複・類似ルールは見つかりませんでした。整理は不要です。」と表示して終了する。

### ステップ6: 統合を実行する

承認された統合ごとに以下を順番に実行する:

**6-1. 統合後の内容をユーザーに確認する**

```
## 統合後の内容（{残すルール名}.md）

{統合後の Markdown 内容}

この内容で更新しますか？
  [yes]  この内容で実行する
  [no]   この統合をスキップする
  [edit] 内容を修正して再提示する
```

- `no` の場合: この統合をスキップして次へ
- `edit` の場合: ユーザーの修正指示を反映して再提示する

**6-2. ファイル操作を実行する**

1. 残すルールファイルを Write で更新する
2. 削除するルールファイルを Bash で削除する
3. CLAUDE.md から削除したルールの `@rules/{name}.md` 行を Edit で削除する
4. 削除したルールの clusters JSON ファイルを Bash で削除する

### ステップ7: コミット

ルールファイルまたは CLAUDE.md に変更があった場合のみコミットする。

> **注意:** clusters JSON の削除のみの場合はコミット不要。
> `.claude/instincts/clusters/` は `.gitignore` 対象のため git 管理外。

コミットメッセージ形式:

```
chore: prune-rules - {操作の概要}

{実行した操作の一覧}

[en-sync: n/a]
```
