# /prune-reports スキル

`.claude/reports/` に蓄積したレポートファイルを整理する。
種別ごとに直近N件を残し、古いものをユーザーの確認後に削除する。

## デフォルト保持件数

| 種別 | 保持件数 |
|---|---|
| requirements-report | 3件 |
| architecture-report | 3件 |
| plan-report | 3件 |
| test-report | 5件（イテレーションが多いため多め） |
| code-review-report | 3件 |
| security-review-report | 3件 |

## 実行手順

### ステップ1: レポートファイルを収集する

`Glob` で `.claude/reports/*.md` を全件取得する。

ファイルが0件の場合は「レポートファイルが見つかりませんでした」と表示して終了する。

### ステップ2: 種別ごとにグループ化する

ファイル名のプレフィックス（日付より前の部分）で種別を判別する。

例:
- `requirements-report-20260411-123456.md` → 種別: `requirements-report`
- `test-report-20260410-093000.md` → 種別: `test-report`

上記以外のプレフィックスのファイルはユーザー作成の可能性があるため**対象外**とする。

### ステップ3: 削除対象を決定する

各種別のファイルをファイル名の降順（新しい順）にソートし、
デフォルト保持件数を超えた古いファイルを削除候補とする。

### ステップ4: ユーザーに確認する

削除候補がない場合は「整理不要です（全種別が保持件数以内です）」と表示して終了する。

削除候補がある場合は以下の形式で提示する:

```
## /prune-reports 確認

以下のレポートを削除します（種別ごとに直近N件を残します）:

### requirements-report（3件保持 → 1件削除）
- requirements-report-20260401-100000.md

### test-report（5件保持 → 3件削除）
- test-report-20260401-090000.md
- test-report-20260401-083000.md
- test-report-20260331-150000.md

合計 N 件を削除します。実行しますか？
  [yes] 削除する
  [no]  キャンセルする
```

### ステップ5: 削除を実行する

yes の場合: 削除候補のファイルを Bash で削除する。

```bash
rm .claude/reports/{ファイル名}
```

削除完了後、「N件のレポートを削除しました」と報告する。

> **注意:** `.claude/reports/` は `.gitignore` 対象のため、コミットは不要。
