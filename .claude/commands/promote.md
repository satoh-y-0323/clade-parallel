# /promote コマンド

プロジェクト固有のスキルまたはルールを、
グローバル（~/.claude/）に展開して全プロジェクトで使えるようにする。

## 実行手順
1. プロジェクト固有ファイルを一覧表示する:
   - Glob ツールで `.claude/skills/project/*.md` を検索する
   - Glob ツールで `.claude/rules/**/individual/*.md` を検索する
   - Read ツールで `.claude/settings.json` の `mcpServers` キーを確認する
   - 結果を整形して出力する

2. 以下の形式で候補を提示する:
   ```
   ## グローバル展開候補

   ### スキル
   1. {ファイル名}: {説明の最初の1行}
   2. ...

   ### ルール
   1. {ファイル名}: {説明の最初の1行}
   2. ...

   ### MCP サーバ
   1. {サーバ名}: {対応するスキルファイルの説明1行 or "スキルファイルなし"}
   2. ...

   展開するものを番号で選んでください。
   ※汎用化が必要なプロジェクト固有の記述は自動的に除去します。
   ```

3. 選択されたファイル・設定を汎用化する:
   - プロジェクト名・固有パス・固有変数名を汎用的な表現に置換する
   - MCP サーバが選択された場合は、settings.json のエントリ内容を確認してユーザーに提示する
   - 汎用化した内容をユーザーに確認する

4. グローバルに保存（ユーザーが承認した場合のみ実行）:
   - スキル → `~/.claude/skills/{name}.md`
   - ルール → `~/.claude/rules/{category}/{name}.md`
   - MCP サーバ → `~/.claude/settings.json` の `mcpServers` に追記 ＋ スキルファイルを `~/.claude/skills/{name}-mcp.md` にコピー

5. 展開結果を報告する:
   ```
   ## 展開完了
   - {ファイル名} → ~/.claude/skills/{name}.md
   次回セッションから全プロジェクトで使用可能になります。
   ```

## 昇格基準の目安
- 2つ以上の異なるプロジェクトで同じ手順・設定が有効だった
- プロジェクト固有の名称・パスを含まずに記述できる
- 他の開発者にも有効と思われる

## MCP サーバ昇格時の注意
- 環境変数（APIキー等）が必要なサーバは、昇格後も各プロジェクトで `.env` または `settings.local.json` への設定が別途必要
- `~/.claude/settings.json` への追記後、Claude Code の再起動が必要

## 注意
このコマンドはプロジェクトレベルの設定（GitHubテンプレート）としても機能する。
グローバル展開はユーザーの任意操作であり、自動実行はしない。
