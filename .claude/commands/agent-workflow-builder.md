# /agent-workflow-builder コマンド

ユーザーの業務をヒアリングし、その業務に特化したエージェント群を自動生成するメタエージェント。親 Claude が Q&A とワークフロー設計を担当し、サブエージェントがファイル生成を担当する。

## 親 Claude の責務

このコマンドは親 Claude が直接実行する。サブエージェントはファイル生成フェーズ（Phase 3 以降）のみを担当する。

## 実行フロー

### Step 1: 上流レポートの読み込み（再開チェック）

Glob で `.claude/reports/workflow-report-*.md` を検索する:
- **存在する場合**: 最新ファイルを Read して Phase 3 から再開する（Step 4 へジャンプ）
- **存在しない場合**: Phase 1 から開始する（Step 2 へ進む）

### Step 2: Q&A の実施（Phase 1〜2）

#### Phase 1: ヒアリング

以下の質問を順番にテキストで出力してユーザーの回答を待つ（1問ずつ出力し、回答を受けてから次へ進む）。

**Q1: 職種・仕事の内容**

```
どんな仕事をされていますか？
職種や担当業務を簡単に教えてください。
（例: 営業事務、マーケター、人事担当、プロジェクトマネージャー など）
```

**Q2: 繰り返し作業**

```
毎日または毎週、繰り返している作業を教えてください。
「毎週月曜に売上を集計して上司にメールする」のような具体的な作業が理想です。
複数ある場合は、一番自動化したいものを選んでください。
```

**Q3: インプット（作業の開始点）**

```
その作業は何から始まりますか？「インプット」となるものを教えてください。
（例: Excel ファイル、受信メール、Slack の通知、顧客からの依頼 など）
```

**Q4: アウトプット（作業の終点）**

```
作業が完了したとき、何を「成果物」として渡しますか？
（例: 報告書をメールで送る、スプレッドシートを更新する、資料を保存する など）
```

**Q5: 一番大変なステップ**

```
その作業の中で、一番時間がかかる・大変だと感じるところはどこですか？
（例: 情報を集めてまとめる部分、文章を考える部分、確認・チェックの部分 など）
```

**Q6: 確認・承認のタイミング（任意）**

```
誰かに確認・承認をとるタイミングはありますか？
（例: 上司に内容を確認してから送る、チームに共有してフィードバックをもらう など）
なければ「なし」と答えてください。
```

#### Phase 2: ワークフロー設計と確認ループ

Phase 1 の回答をもとにワークフローを設計し、以下の形式でユーザーに提示する。
「yes」が得られるまで修正・再提示を繰り返す:

```
ヒアリング内容をもとに、以下のワークフローを提案します。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
作業名: {作業名（例: 週次売上報告ワークフロー）}

  Step 1. [収集] agent-{name}     → {このステップでやること}
  Step 2. [作成] agent-{name}     → {このステップでやること}
  Step 3. [確認] agent-{name}     → {このステップでやること}
  ...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

このワークフローで合っていますか？
調整したい点があればお知らせください。（例: Step 3 はいらない、Step 2 と 3 を一緒にしたい）
問題なければ「yes」と答えてください。
```

ユーザーから修正指示が来た場合はワークフローを更新して再提示する。

**ステップの割り当て基準:**

| 状況 | 役割カテゴリ | エージェント名（例） |
|------|------------|------------------|
| 情報の収集・整理が必要 | `収集` | agent-gatherer |
| 文章・資料・データの生成が必要 | `作成` | agent-creator |
| 内容の確認・品質チェックが必要 | `確認` | agent-checker |
| 承認フローや最終調整が必要 | `仕上` | agent-finalizer |
| 外部への送信・共有が必要 | `配布` | agent-distributor |

**ステップ数の目安**: 2〜5 ステップ（シンプルな業務は 2〜3、複雑な業務は 4〜5）

エージェント名はユーザーの業務内容に合わせた名前にする（汎用的な名前より業務を表す具体的な名前を優先）。

### Step 3: Q&A 結果の整理

Phase 1〜2 の結果を以下の構造に整理する:
- 職種・仕事の内容
- 繰り返し作業の内容
- インプット・アウトプット
- 一番大変なステップ
- 確認・承認のタイミング
- 承認済みワークフロー（作業名・各ステップのエージェント名・役割・やること）
- 統括コマンド名

### Step 4: サブエージェントの一発起動

Agent ツールで `subagent_type: workflow-builder` を指定して起動する。プロンプトに以下を含める:

```
## 作業依頼
承認済みワークフローのエージェントファイル生成と CLAUDE.md 更新

## workflow-report のパス（再開時のみ）
{ファイルパス または「なし（新規生成）」}

## ユーザーとの Q&A 結果

### Phase 1: ヒアリング
Q1（職種・仕事）: {回答}
Q2（繰り返し作業）: {回答}
Q3（インプット）: {回答}
Q4（アウトプット）: {回答}
Q5（一番大変なステップ）: {回答}
Q6（確認・承認）: {回答}

### Phase 2: 承認済みワークフロー
作業名: {作業名}
統括コマンド名: {コマンド名}

| Step | エージェント名 | 役割カテゴリ | やること |
|------|--------------|------------|---------|
| 1    | agent-{name} | {カテゴリ}  | {やること} |
| 2    | agent-{name} | {カテゴリ}  | {やること} |

## 出力指示
1. workflow-report を `.claude/reports/workflow-report-*.md`（write-report.js 経由）に保存すること
2. 各ステップごとに以下の 2 ファイルをセットで生成すること（Write ツール使用可）:
   - `.claude/commands/agent-{name}.md`（親 Claude 用の Q&A コマンドファイル）
   - `.claude/agents/{name}.md`（サブエージェント定義ファイル）
3. 統括コマンドを `.claude/commands/{workflow-name}.md` に生成すること（Write ツール使用可）
4. `CLAUDE.md` の `## User Agents` セクションに追記すること（Edit ツール使用可）
5. 最終メッセージに生成したファイル一覧を含めること
- AskUserQuestion / SendMessage は使わないこと
- ファイル生成後は終了すること（報告は親 Claude が担当）
```

否認後の再生成時はプロンプトに以下を追加する:
```
## 再生成モード
- ユーザーからの修正指示: {指示内容}
```

### Step 5: 結果の受け取り

サブエージェントの最終出力から生成されたファイル一覧を確認する。

### Step 6: 承認確認

ユーザーに以下をテキストで提示する:

```
ワークフロービルダーが完了しました。

生成したファイル:
  - .claude/commands/agent-{name}.md × {N} 件（親 Claude 用コマンド）
  - .claude/agents/{name}.md × {N} 件（サブエージェント定義）
  - .claude/commands/{workflow-name}.md（統括コマンド）
  - CLAUDE.md 更新済み

使い方:
  /{workflow-name} を実行すると、{作業名} のフローが自動で動き始めます。

修正が必要な場合はお知らせください。
問題なければ「ok」と答えてください。
```

### Step 7: 承認記録

workflow-builder は固定のレポートファイルがないため、承認記録は省略する。

### Step 8: 否認時の再起動

修正が必要な場合、修正指示を含めた新プロンプトで Step 4 から繰り返す。

---

## 用途
- 業務ヒアリングからカスタムエージェント群を自動生成する
- 繰り返し作業を Clade エージェントとして定義する

## 注意
- 標準ワークフロー（フェーズ構成）とは独立して動作する
- このコマンドは単独で完結する（他エージェントへの連携は不要）
- 生成されるエージェントテンプレートは新方式（親子分担・一発起動型）に従わせる
