# /init-session コマンド

セッションを初期化し、前回の作業状態を復元する。
ユーザーがセッション開始時に手動で実行する。

## 実行手順
1. 本日の日付を確認する
2. Glob ツールで `.claude/settings.local.json` を検索し、ファイルが存在しない場合は以下の警告をユーザーに表示する:

   > ⚠️ セットアップ未実行の警告
   >
   > このリポジトリはセットアップが完了していません。
   > Claude Code を正しく利用するには、先にセットアップスクリプトを実行してください。
   >
   > 実行方法:
   >   Linux / macOS : bash setup.sh
   >   Windows       : powershell -File setup.ps1
   >
   > セットアップを行わない場合の影響:
   >   - settings.local.json が配置されず、並列エージェント（worktree）が動作しません
   >   - テンプレートファイルが残ったままになり、意図しない設定が適用される可能性があります

3. Bash ツールを **2回に分けて** 以下を別々に実行する（`&&` で結合しないこと・セキュリティ対策）:
   - 1回目: `node .claude/hooks/clear-file-history.js`
   - 2回目: `node .claude/hooks/enable-sandbox.js`
4. 最新セッションファイルを読み込む:
   Glob ツールで `.claude/memory/sessions/*.tmp` を検索し、
   ファイル名（YYYYMMDD）が最大のものを選択して Read ツールで読み込む。
5. `.claude/memory/memory.json` を Read ツールで読み込む（存在する場合）
6. `.claude/instincts/clusters/` の全JSONファイルを Glob ツールで検索し、
   存在する場合は Read ツールで読み込む
7. 以下の形式でサマリを表示する:

---
## セッション再開 ({今日の日付})

### 前回セッション ({前回の日付})
**残タスク:**
{残タスクリスト}

**前回うまくいったこと:**
{成功アプローチの要約}

**注意事項（前回失敗したこと）:**
{失敗アプローチの要約}

### 有効なインスティンクト
{プロジェクト固有インスティンクト数}件のパターンが読み込まれました

---

続きから作業しますか？それとも新しいタスクを開始しますか？

8. ユーザーの回答に応じて分岐する:

   **「続きから」の場合**: 残タスクの内容を確認してそのまま作業に入る。トリアージは行わない。

   **「新しいタスク」の場合**: 以下のタスク規模確認を行う:

   ```
   どんな作業を予定していますか？
     [small]  小さい修正・調査・質問
     [medium] 機能追加・バグ修正
     [large]  新機能設計・大規模変更
   ```

   選択に応じて以下を案内する:

   | 選択 | 具体例 | 案内内容 |
   |---|---|---|
   | small | 設定値の変更、ドキュメント修正、調査・質問 | 「直接指示してください。エージェントは不要です」 |
   | medium | 新しい処理の実装、バグの調査・修正 | 「`/agent-developer` から始めましょう」 |
   | large | 新モジュールの設計、既存設計の大幅変更 | 「フルワークフロー（`/agent-interviewer` から）を使いましょう」 |
