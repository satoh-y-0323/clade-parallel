# ADR-003: Defer worktree isolation to v0.3; rely on timestamped filenames in v0.1

- **Date:** 2026-04-20
- **Status:** Accepted

## Context

複数の Clade エージェントを並列実行すると、各エージェントが同じ `.claude/reports/` ディレクトリにレポートファイルを書き出す。
同じ作業ディレクトリ（cwd）を共有するため、ファイル名が衝突するリスクがある。

衝突回避の方法として以下を検討した:

1. 同じ cwd + タイムスタンプ命名（`YYYYMMDD-HHmmss`）に依存する
2. git worktree で各エージェントを隔離し、完了後にマージする
3. タスクごとに個別の cwd を用意する

## Decision

v0.1 では同じ cwd を使い、Clade の既存のタイムスタンプ命名規則（`code-review-report-YYYYMMDD-HHmmss.md` 等）に衝突回避を委ねる。

worktree 隔離は v0.3 以降に延期する。

## Rationale

| 観点 | 理由 |
|---|---|
| MVP の複雑度を下げる | worktree の作成・マージは実装コストが大きく、MVP の要件を満たすには不要 |
| 既存運用の延長 | Clade エージェントはすでにタイムスタンプ付きファイル名でレポートを出力している。この命名規則が衝突回避として実際に機能している |
| 実用上の衝突リスクは低い | 各エージェントは起動タイミングが数秒ずれる（プロセス起動コスト）うえ、実行時間も異なるため、秒単位の衝突が起きる可能性は極めて低い |

## Alternatives Considered

**worktree 隔離**: 完全な分離が保証されるが、worktree の作成・チェックアウト・マージのロジックが必要。エラーハンドリングも複雑になる。v0.3 の機能として設計を温存する。

**タスクごと個別 cwd**: `.claude/reports/` の位置がタスクごとにバラバラになり、ユーザーが各ディレクトリを確認しなければならなくなる。ユーザー体験が悪化する。

## Consequences

- `Task.cwd` フィールドを設計に持たせており、v0.3 で worktree パスを差し込むだけで隔離に対応できる構造としている
- v0.3 で worktree 隔離を導入する際、マニフェストの `writes:` 宣言（ADR-005 参照）と組み合わせてファイル衝突を静的チェックできるようにする計画
- 稀なケース（秒以内の同時起動）でタイムスタンプが衝突した場合、後から書いた側が上書きする。v0.1 ではこれを許容する
